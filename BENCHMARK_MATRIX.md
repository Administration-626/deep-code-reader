# Deep Code Reader v2: Continuous Cognition Benchmark Matrix

To ensure the protocol maintains state integrity and prevents "cognition rot" over time, any implementation of `deep-code-read` (whether a standalone Python CLI, an Aider script, or an orchestrated agent system) MUST pass this 4-layer Benchmark Matrix.

This suite is not about testing whether "the AI is smart." It tests whether the **Incremental Orchestration** and **State Replacement Semantics** behave like a robust database transaction under extreme edge cases.

---

## Layer 1: Invalidation Correctness
**Goal:** Prove the system invalidates exactly what it should. False Positives (over-invalidation) cost tokens. False Negatives (missed invalidation) cause silent cognition rot.

### Test 1A: The Leaf Node Change
- **Action:** Modify an isolated file (e.g., `src/cli/parser.rs`) that no other module imports.
- **Assertion:** 
  - ONLY the `cli` module is marked `dirty` with `direct_file_change`.
  - Its `verification_pass_rate` and `module_hash` are overwritten.
  - The `verified_at` for all other modules must remain strictly unchanged.

### Test 1B: The Shared Contract Avalanche
- **Action:** Modify a core type definition (e.g., `src/shared/types.rs`) imported by 3 other modules.
- **Assertion:** 
  - The `shared` module AND all 3 importing modules are marked `dirty`.
  - All 4 modules must undergo a complete Phase 4 ABC verification loop.
  - If any importing module is skipped, the test FAILS (Critical False Negative).

### Test 1C: Module Deletion & Ghost Cleanup
- **Action:** Delete an entire directory mapped to a module.
- **Assertion:** 
  - The module disappears from `.metadata.json` entirely. No "orphan" or "ghost" metadata remains.
  - Phase 5 `SKILL.md` regenerates the dependency graph and completely removes the deleted module from cross-module scenarios.

## Layer 2: Metadata Integrity
**Goal:** Prove `.metadata.json` is deterministic, hermetic, and immune to append-only poisoning.

### Test 2A: The Idempotent Rerun
- **Action:** Run Phase 0 on a codebase with NO changes (`git diff` is empty).
- **Assertion:** 
  - The system safely exits.
  - `.metadata.json` and `SKILL.md` byte-for-byte match the previous state. No timestamp drift occurs unless an actual regeneration happened.

### Test 2B: Ghost Gap Prevention
- **Action:** Take a module with 15 `open_gaps`. Make a trivial whitespace change. Run incremental rerun. Assume the Agent scores 100% locally.
- **Assertion:** 
  - The `open_gaps` array is strictly replaced with `[]`.
  - No old gaps are merged or appended to the new array. 
  - *Note: This proves strict Replacement Semantics, though it highlights the "Confidence Inflation" problem addressed in Layer 3.*

---

## Layer 3: Verification Economics
**Goal:** Measure the actual token and time cost of proving cognition validity.

### Test 3A: Plateau Detection Speed
- **Action:** Feed an extremely complex compiler module to Agent A. Force Agent C to answer questions it cannot possibly know.
- **Assertion:** 
  - The system must trigger the `plateau` or `max_rounds` stop reason within 3 rounds.
  - The system must not enter an infinite loop. 
  - The cost (tokens/round) must be recorded to calculate `avg_tokens_per_verified_fact`.

### Test 3B: Confidence Tri-Layer Separation
- **Action:** Trigger an incremental update on a single line of code in a massive module.
- **Assertion:** 
  - The `local_verification_confidence` reaches 100%.
  - The `historical_coverage_confidence` REMAINS LOW or unchanged, explicitly preventing the system from hallucinating that it has mastered the entire module's complexity based on a 1-line diff.

---

## Layer 4: Longitudinal Stability (The Ultimate Test)
**Goal:** Prove the system can survive continuous iteration without degrading into a hallucinated mess.

### Test 4A: The 100-Commit Marathon
- **Action:** Script an automation that replays 100 sequential commits from an active, real-world repository (e.g., Kubernetes, ActPlane). Run the incremental protocol after each commit.
- **Assertion:** 
  - After 100 runs, compare the incrementally generated `SKILL.md` to a `SKILL.md` generated by running a **Full (Cold) Generation** on the 100th commit directly.
  - The architectural topologies, dependency graphs, and module summaries must be semantically coherent and free of "stale residue" from commit #10.
